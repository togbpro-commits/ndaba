# Agent Instructions

## Package Manager
- Use **npm** (lockfile: `package-lock.json`)

## Commands
| Task | Command |
|------|---------|
| Dev server | `npm run dev` |
| Production build | `npm run build` |
| Lint | `npm run lint` |
| Type check | `node scripts/check-errors.js` (writes `error.md`) |

## Architecture
- **Next.js 16 App Router** — all routes live in `src/app/`
- **All pages are client components** (`'use client'` directive required)
- **Path alias**: `@/*` → `./src/*`
- **Tailwind CSS v4** via `@tailwindcss/postcss` — no `tailwind.config.js`; theme tokens defined in `src/app/globals.css` using `@theme inline`

## Auth (custom, not real Clerk)
- `src/lib/clerk.tsx` is a **custom React context** — NOT `@clerk/nextjs` middleware/route protection
- Session stored in `localStorage` under `ndaba_auth_session`
- Demo credentials: `test` / `test` (admin), `admin@ndabasattorneys.co.za` / `1234`, `staff@ndabasattorneys.co.za` / `4321`
- Hooks: `useUser()`, `useAuth()` — both require `<ClerkProvider>` wrapper (already in root layout)

## Database (Supabase with mock fallback)
- `src/lib/db.ts` exports a unified `db` object with `Lead` and `Case` interfaces
- If `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are set → real Supabase
- Otherwise → falls back to `localStorage` mock data (`ndaba_leads`, `ndaba_cases` keys)
- `db.isMock` flag indicates which mode is active
- All CRUD operations: `getLeads`, `insertLead`, `updateLeadStatus`, `deleteLead`, `getCases`, `insertCase`, `updateCase`, `deleteCase`

## Conventions
- No test suite configured — verify via `node scripts/check-errors.js` and `npm run lint`
- `error.md` is auto-generated by `scripts/check-errors.js` — do not edit by hand
- Fonts: `Playfair_Display` (serif headings) + `Plus_Jakarta_Sans` (body) via `next/font/google`
- Dark mode uses `.dark` class toggle with CSS custom properties
